# Плагины
Плагины служат для добавления команд, которых изначально нет в программе.


## Создание
1. Создайте папку в plugins.
2. Внутри создайте manifest.json:
```json
{
	"name": "имя",
	"description": "описание",
	"version": "версия",
	"type": "тип",
	"commands": {
		"команда": "данные"
	}
}
```
3. Готово.

Этих действий достаточно для определения папки как плагина. Реализация команд зависит от типа плагина. Инструкции по созданию для разных типов в разделе [Типы плагинов](#типы-плагинов).


## Манифест

- Name - Отображаемое имя плагина.
- Description (Необязательно) - Описание плагина.
- Version - Отображаемая версия плагина.
- Type - Определяет обработчик плагина.  
Подробнее в [Типы плагинов](#типы-плагинов).
- Commands - Словарь, где ключи используются при распознавании, для указания нескольких команд используйте разделитель `|` например:
`"запусти браузер|открой браузер"`.  
Данные под ключами задают поведение команды и зависят от [типа](#type) плагина.

### Пример для типа `program`
```json
{
	"name": "МойПлагин",
	"description": "Описание плагина",
	"version": "1.0",
	"type": "program",
	"commands": {
		"запусти браузер|открой браузер": "scripts/open_browser.bat",
		"закрой браузер": "scripts/close_browser.bat"
	}
}
```


## Возврат команд
В типах с возвратом данных возможно вернуть json-строку в формате:
```json
{
	"action": "тип",
	"speak": "текст",
	"context": "данные"
}
```

- Action - тип действия программы (None/Close)
- Speak - текст который будет озвучен пользователю
- Context - следующий распознанный текст будет передан в обход фразы активации и поиска команд

## Типы плагинов
Для каждого типа используется свой обработчик, который определяет логику обработки данных команд. Например, в типе `program` данные - это пути к исполняемым файлам.
